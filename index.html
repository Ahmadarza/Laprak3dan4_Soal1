A. Penjelasan Konsep Dasar PHP

a. Jelaskan apa itu PHP.

PHP (Hypertext Preprocessor) adalah bahasa pemrograman yang dirancang khusus untuk pengembangan aplikasi web. PHP biasanya digunakan untuk membuat situs web dinamis dan memiliki kemampuan untuk menyisipkan kode langsung ke dalam file HTML, sehingga memudahkan pengembang web untuk menggabungkan kode pemrograman dengan markup.

b. Sebutkan keuntungan PHP dibandingkan dengan bahasa pemrograman lainnya dalam konteks pengembangan web.

  • Mudah Dipelajari dan Digunakan: PHP dirancang untuk pengembangan web, sehingga sintaksnya sederhana dan mudah dipahami.
  • Cross-Platform: PHP bisa dijalankan di berbagai platform seperti Windows, Linux, dan macOS, Menjadikannya fleksibel dan dapat diimplementasikan di lingkungan server yang berbeda.
  • Open Source: PHP adalah perangkat lunak open-source, sehingga tidak memerlukan biaya lisensi dan dapat digunakan oleh siapa saja.
  • Kinerja yang baik: PHP dapat menangani aplikasi web dengan lalu lintas tinggi.
  • Integrasi yang Mudah: PHP kompatibel dengan berbagai jenis database seperti MySQL dan dapat dengan mudah diintegrasikan dengan teknologi web lain seperti HTML, CSS, dan JavaScript.
  • Stabil dan Teruji.

B. Perbandingan GET dan POST

a. Jelaskan perbedaan antara metode GET dan POST dalam penggunaan di sebuah website.

  • Perbedaan GET:
    • History: Parameter/nilai tersimpan di browser dan bisa diakses ulang karena nilai menjadi bagian dari URL.
    • Bookmark: Parameter bisa dibookmark.
    • Tombol Kembali/Kirim Ulang: Halaman sebelumnya langsung ditampilkan tanpa mengeksekusi ulang karena tersimpan dalam cache browser.
    • Parameters: Hanya bisa mengirimkan parameter melalui URL.
    • Keamanan: Lebih terbuka karena data terlihat.
    • Penggunaan: Digunakan lebih untuk kebutuhan mengambil data.
    • Besar Data yang Dikirim: Memiliki batasan panjang.
  • Perbedaan POST:
    • History: Parameter/nilai tidak tersimpan di browser.
    • Bookmark: Parameter/nilai tidak dapat dibookmark.
    • Tombol Kembali/Kirim Ulang: Browser biasanya memberi peringatan/menanyakan apakah data akan dikirim ulang.
    • Parameters: Dapat mengirimkan parameter URL sekaligus mengirimkan data tersembunyi ke server.
    • Keamanan: Lebih sulit diakses karena data tersembunyi.
    • Penggunaan: Digunakan untuk mengirim data yang lebih rahasia seperti password.
    • Besar Data yang Dikirim: Tidak memiliki batasan yang ketat seperti GET.

b. Berikan contoh kasus penggunaan yang tepat untuk masing-masing metode.

  • GET: Pencarian di Google, Menampilkan Daftar Produk, Filter atau Sortir Data.
  • POST: Login atau Registrasi Akun, Mengisi Formulir Online, Melakukan Pembelian, Upload File.

C. Penggunaan Include, Require, dan Koneksi PHP dengan MySQL

a. Jelaskan fungsi include dan require dalam PHP, termasuk perbedaan mendasar antara keduanya.

Include Jika file yang disertakan tidak ditemukan atau terjadi kesalahan, PHP akan memberikan warning tetapi tetap melanjutkan eksekusi skrip. Dan Require Jika file yang disertakan tidak ditemukan atau terjadi kesalahan, PHP akan menghasilkan fatal error dan menghentikan eksekusi skrip.

b. Berikan contoh implementasi sederhana untuk menyambungkan PHP dengan database MySQL.

                        <?php
                        $host = "localhost"; 
                        $username = "root";  
                        $password = "";      
                        $dbname = "test_db"; 
                
                        $conn = new mysqli($host, $username, $password, $dbname);
                
                        if ($conn->connect_error) {
                            die("Koneksi gagal: " . $conn->connect_error);
                        }
                        echo "Koneksi berhasil";
                
                        $conn->close();
                        ?>

  • Modularitas Kode
  • Pengurangan Redundansi
  • Peningkatan Keamanan
  • Peningkatan Pemeliharaan Kode
  • Reusabilitas Kode
